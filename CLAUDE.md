# 记账APP - 产品文档 & 开发约定

## 一、项目概述

| 项目 | 内容 |
|------|------|
| **APP名称** | 记账 |
| **平台** | Windows + macOS 桌面应用 |

## 二、核心功能

### 2.1 记一笔花销
- 输入金额（人民币，支持小数点后两位，如 12.50）
- 选择日期（默认当天）
- 选择分类（一级大类 → 二级小分类）
- 可选备注（文字说明）

### 2.3 查看账单
- 按日期查看（今天、本周、本月、自定义日期范围）
- 按分类查看（某个大类或小类的支出汇总）
- 支出统计（总金额、日均支出、分类占比饼图）

### 2.4 数据管理
- 所有数据保存在本地电脑
- 支持导出（Excel或CSV格式）
- 支持数据备份和恢复

## 三、界面设计原则
- 简洁直观，适合非技术用户
- 操作步骤少，记一笔账不超过3步
- 界面语言为中文
- 视觉风格：清爽、现代

## 四、⚡ 重要开发约定（必须遵守）

### 项目背景
> 本项目的产品方（用户）不懂编程技术，是首次使用AI工具开发软件。

### 技术决策规则
**在整个项目开发过程中，任何涉及技术选型、方案设计、工具选择的决策，都必须遵循以下流程：**

1. **列出方案**：AI 必须列出 2-4 个可行方案
2. **解释说明**：用通俗易懂的大白话解释每个方案是什么、有什么优缺点
3. **给出推荐**：AI 可以给出推荐方案并说明理由
4. **等待决定**：由用户（产品方）来做最终决定，AI 不得擅自选择

### 触发场景示例
- 选择UI框架/组件库
- 选择数据存储方式
- 选择图表库
- 选择打包配置
- 界面布局方案
- 任何"有多种做法"的技术点

### 不适用场景
- 修复明显的程序错误（Bug）
- 代码语法修正
- 用户明确说"你来决定"的事项
